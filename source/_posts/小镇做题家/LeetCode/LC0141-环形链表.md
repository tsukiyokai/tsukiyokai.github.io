## 题目

给定一个链表，判断链表中是否有环。
如果链表中有某个节点，可以通过连续跟踪next指针再次到达，则链表中存在环。为了表示给定链表中的环，我们使用整数pos来表示链表尾连接到链表中的位置（索引从0开始）。如果pos是-1，则在该链表中没有环。注意：pos不作为参数进行传递，仅仅是为了标识链表的实际情况。
如果链表中存在环，则返回true。否则，返回false。
注：这道题描述的很复杂，其实没必要，pos跟你没关系，是服务器构造环形链表用的，写题不用管。

进阶：你能用O(1)内存解决此问题吗？

## 解法1：哈希

逐个遍历列表，将结点地址放在哈希表中。在任何时候，如果达到NULL，则返回false；如果当前节点的下一个指向哈希中先前存储的任何节点，则返回true。

**代码**

```c++
class Solution {
public:
    bool hasCycle(ListNode* head) {
        unordered_set<ListNode*> set;
        while (head) {
            if (set.find(head) != set.end()) return true;
            set.insert(head);
            head = head->next;
        }
        return false;
    }
};
```

**分析**

空间复杂度O(n)

## 解法2：标记位

修改链表结点的数据结构，使其天然包含一个标记位。

```c++
struct Node {
    int data;
    struct Node* next;
    int flag;
};
```

**分析**

空间复杂度O(1)

## 解法3：Floyd判圈算法

Floyd判圈算法，又称龟兔赛跑算法，是一个可以在有限状态机、迭代函数或者链表上判断是否存在环，求出该环的起点与长度的算法。
其实也就是一种双指针。

**代码**

```c++
class Solution {
public:
    bool hasCycle(ListNode* head) {
        if (!head) return false;
        ListNode* slow = head, * fast = head;
        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;
            if (slow == fast) return true;
        }
        return false;
    }
};
```

**注释**

思考一点：为什么`while (fast && fast->next);`中对fast和fast->next的判断不能颠倒？这么做看似充分利用了与门的短路算法，因为fast->next显然比fast更容易到达边界，但是注意，fast和fast->next归根结底是同一个客体，如果某一时刻fast实际上已经为空，再判断fast->next是否为空是没有意义的，正如数学运算中零不能做分母。
1. 空指针不能取其指针域。这个错误已经犯过好多次了，下次不能再犯。
2. 什么时候才能考虑利用短路算法来提高效率？当判断对象之间不是同一客体时再做考虑，否则会有类似上述的风险。
