## 题目

给定一个字符串，请你找出其中不含有重复字符的最长子串的长度。

## 解法 1：暴力

暴力法时间复杂度为 O(n^3)

## 解法 2：滑动窗口

时间复杂度为 O(n^2)

**重复字符**即最多只出现一次，一旦涉及出现次数即想到需要用到哈希表。
又看到关键字**最长子串**，涉及子串，可以考虑滑动窗口。

**代码**

```c++
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        if (s.size() == 0) return 0;
        unordered_set<char> set;
        int maxlen = 0;
        int left = 0;
        for (int right = 0; right < s.size(); right++) {
            while (set.find(s[right]) != set.end())
                set.erase(s[left++]);
            maxlen = max(maxlen, right - left + 1);
            set.insert(s[right]);
        }
        return maxlen;
    }
};
```

**注释**

unordered_set 中实时存放当前窗口内出现的字符。

对 right 的遍历是最主要的遍历，滑动窗口的要点是 right 和 left 只能前进，不能后退，不断从左缩小窗口直到窗口中不存在与右边界重复的字符。

当右端点读到重复字符时，因为不知道原有字符出现在当前左右端点所划定子串中的什么位置，而右端点又只能前进，不能回头去找该重复字符（事实上让右端点折返的做法也非常荒谬），所以此时应不断使左端点前进并丢弃做旧位置上的字符，直到新的左端和此时的右端两个端点所划定的范围中不存在与此时的右端点所指字符重复的字符，此时正式到达新的稳定状态——左右端点划出来一个范围，这个范围表示的字符串不含有任何重复字符。

新的稳态完全不能保证比上一个稳态优异（即更长），因此我们从一个稳态跳向另一个稳态的过程中，还维护着一个表示全程最大长度的变量并不断比较和更新它。

认清一个明显的事实：left 和 right 所框定的子串，不是无重复字符的子串，就是处于正在向无重复字符的子串这个目标不断调整的过程中。

另一个显而易见的要点是，在判断 set 中是否存在 right 指向字符的副本之前，不能先把 right 处的该字符加进 set 里，不然 set 就被污染了，判重失去意义。
