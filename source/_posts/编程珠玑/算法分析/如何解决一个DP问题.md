动态规划是一种在多项式时间内解决某些特定类型问题的技术。
动态规划解的速度比指数暴力法快，并且易于证明其正确性。
在研究如何动态思考问题之前，我们需要了解：
重叠子问题
最佳子结构属性

解决DP的步骤：
```
1. 确定是否是DP问题
2. 写出带有最少参数的状态表达式
3. 写出状态关系
4. 表格化或记忆化
```

## 如何将问题归类为动态规划问题？

通常情况下，所有需要最大化或最小化一定数量的问题或计数问题，即在一定条件下或一定概率下的安排计数问题，都可以用动态规划来解决。

所有的动态规划问题都满足重叠子问题的性质，而大多数经典的动态规划问题也满足最优子结构的性质。一旦我们在给定的问题中观察到这些性质，就要确保它可以用DP来解决。

## 决定状态

DP问题都是关于状态及其转换的。这是必须非常小心地完成的最基本的步骤，因为状态转换取决于所做的状态定义的选择。那么，让我们看看“state”这个词是什么意思。

状态可以定义为一组参数，这些参数可以唯一地标识给定问题中的某个位置或站姿。这组参数应尽可能小，以减少状态空间。

例如：在我们著名的背包问题中，我们用两个参数指标和权重来定义我们的状态，即`DP[index][weight]`。这里`DP[index][weight]`告诉我们，通过从0到具有袋重容量的索引，它可以获得最大利润。因此，这里的参数索引和权重可以唯一地识别背包问题的一个子问题。

因此，我们的第一步将是在确定问题是DP问题之后确定问题的状态。

正如我们所知道的，DP是所有关于使用已计算结果来制定最终结果。
所以，我们的下一步将是找到先前状态之间的关系，以达到当前状态。

## 状态关系

这部分是解决DP问题最难的部分，需要大量的直觉、观察和实践。让我们通过考虑一个示例问题来理解它。

给定3个数字（1,3,5），求出这三个数字任意组合相加得出目标数字的组合数，这三种数字可以任意混合和重复。

如：1,3,5组合出6的方式有8种：
1+1+1+1+1+1
1+1+1+3
1+1+3+1
1+3+1+1
3+1+1+1
3+3
1+5
5+1

```c++
// initialize to -1
int dp[MAXN];

// this function returns the number of arrangements to form 'n'
int solve(int n) {
    // base case
    if (n < 0) return 0;
    if (n == 0) return 1;

    // checking if already calculated
    if (dp[n] != -1) return dp[n];

    // storing the result and returning
    return dp[n] = solve(n - 1) + solve(n - 3) + solve(n - 5);
}
```
