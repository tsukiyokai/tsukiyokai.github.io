# 双链表 删除

## 删除具体结点

删除问题按删除位置可归为三类：删头，删尾，删中间结点。如果已知待删除结点和头结点，上述三类问题均可以用两步处理。

1. 如果是头结点被删，需要重新改变头指针
2. 结点被删除后，前后结点要重新链接

算法：

如果头结点是被删除结点，则重新使头指针指向被删除结点的后继结点。（暂且只做这一步）

如果被删除结点不是第一个，说明前面还有结点，前指后。

如果被删除结点不是最后一个，说明后面还有结点，后指前。

```c++
void deleteNode(Node** head_ref, Node* del)
{
    if (*head_ref == NULL || del == NULL)
        return;

    if (*head_ref == del) /*A*/
        *head_ref = del->next;

    if (del->prev != NULL) /*B*/
        del->prev->next = del->next;

    if (del->next != NULL) /*C*/
        del->next->prev = del->prev;

    free(del);
    return;
}
```

解释一下，这段代码逻辑其实类似甘氏图：

A的情境：精确删头。

B的情境：待删除结点的左邻域不为空，即中间结点或尾结点，此时可以从前面的结点伸出next指向后面（如果是中间结点则del.next为后继结点，如果是尾结点则del.next为NULL）。

C的情境：待删除结点的右邻域不为空，即中间结点或头结点，此时可以从后面的结点伸出prev只想前面（如果是中间结点则del.prev为前驱结点，如果是头结点则del.prev为NULL）。

综上所述，传入结点是头结点会走A+C这条路，传入中间结点走B+C，传入尾结点走B。这种巧妙写法同时具有逻辑上的美感和形式上的工整，应学习品味。

![dll](https://media.geeksforgeeks.org/wp-content/cdn-uploads/gq/2014/03/DLL1.png)

为什么传入尾结点与其他两者不太一样？观察上图容易发现，头结点和中间结点均有4条相关指针，而尾结点只有3条相关指针。当结点D准备被删除时，存放在类D中的D->prev和D->next都会自由，紧接着C成为事实上的新任尾结点，C->next=D->next=NULL，（因此时D->prev和D->next只是断开，还没有物理上清空，因此D->next的内容仍然是NULL），直到free(D)，至此，与D相关的一切（data、prev、next）才都被抹去。
