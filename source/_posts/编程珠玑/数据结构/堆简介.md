## 堆 heap

### 堆的概念

n 个元素序列 { k1, k2, k3, k4, k5, k6, …, kn } 当且仅当满足以下关系时才会被称为堆：
`ki <= k2i, ki <= k2i+1 或 ki >= k2i, ki >= k2i+1 (i = 1, 2, 3, 4, …, n/2)`
如果数组的下标是从 0 开始，那么需要满足：
`ki <= k2i+1, ki <= k2i+2 或者 ki >= k2i+1, ki >= k2i+2 (i = 0, 1, 2, 3, …, n/2)`

比如 { 1, 3, 5, 10, 15, 9 } 这个序列就满足 [1 <= 3; 1 <= 5], [3 <= 10; 3 <= 15], [5 <= 9] 这 3 个条件，这个序列就是一个堆。

所以堆其实是一个序列或数组，如果这个序列满足上述条件，那么就把这个序列看成堆。另外，堆又被称作堆积树。

### 堆的定义

定义：一棵完全二叉树，其每个节点都有一个值。特点是根节点值最小（或最大），且根节点的两个子树也是一个堆。根节点值最小的堆称为小根堆，根节点值最大的堆称为大根堆。
学科：计算机科学技术 理论计算机科学 数据结构
来源：计算机科学技术名词第三版

## 二叉堆 binary heap

堆的实现通常是通过构造二叉堆，因为二叉堆应用很普遍，当不加限定时，堆通常指的就是二叉堆。

### 二叉堆的数组表示

二叉堆是一棵完全二叉树。二叉堆通常表示为数组的形式。表示方法如下：

1. 根节点位于 arr[0]
2. 第 i 个节点的相关节点可以表示为：
   父节点：`arr[(i-1)/2]` 向下取整
   左孩子：`arr[2*i+1]`
   右孩子：`arr[2*i+2]`

把堆用数组表示所使用的方法是进行层序遍历。

二叉堆总是满足以下两种堆序性（heap order）之一：
最大堆：父节点的键值总是大于等于任何一个子节点的键值。
最小堆：父节点的键值总是小于等于任何一个子节点的键值。

### 二叉堆的应用

1. 堆排序。堆排序使用二叉堆在 O(nlogn)时间内对数组进行排序。
2. 优先级队列。使用二叉堆可以有效地实现优先级队列，因为它支持在 O(logn)时间内执行 insert()、delete()和 extractmax()、decreaseKey()等操作。二项式堆和斐波那契堆是二叉堆的变体。用堆实现的优先级队列主要用于 Prim 算法和 Dijkstra 算法等图算法中。
3. 图的算法。优先级队列特别适用于 Dijkstra 最短路径和 Prim 最小生成树等图算法。
4. 顺序统计量（Order Statistics）。使用堆可以有效地解决许多问题，如：
   a）数组中第 K 大的元素。
   b）对基本有序的数组进排序。
   c）合并 K 个有序数组。

### 二叉堆的基本操作

| 操作    | 描述                         | 时间复杂度         |
| ------- | ---------------------------- | ------------------ |
| build   | 创建一个空堆                 | O(n)\*             |
| insert  | 向堆中插入一个新元素         | O(logn)            |
| update  | 提升新元素使其符合堆的性质   | O(logn)            |
| get     | 获取当前堆顶元素即根节点的值 | O(1)               |
| delete  | 删除堆顶元素                 | O(logn)            |
| heapify | 使删除堆顶元素的堆再次成为堆 | O(logn)            |
| union   | 将两个堆合并成一个有效堆     | O(n), O(klogn)\*\* |

注：
构造堆的时间复杂度为 O(n)的相关证明参见：[How can building a heap be O(n) time complexity?](https://stackoverflow.com/questions/9755721/how-can-building-a-heap-be-on-time-complexity)
\*\*k 是两个堆之间较小那个堆的长度

某些堆实现还支持其他的一些操作，如斐波那契堆支持检查一个堆中是否存在某个元素。

为了简洁起见，下面以最小堆的相关操作为例进行说明：

1. getMini()：返回最小堆的根节点。时间复杂度为 O(1)。
2. insert()：插入新节点需要 O(logn)时间。我们在完全二叉树最下层的末尾添加一个新键。如果新键大于其父键，则不需要执行任何操作。否则，需要遍历来修复冲突的 heap 属性。
3. delete()：删除节点也需要 O(logn)时间。因为删除操作是通过调用 decreaseKey()将待删除的节点替换为负无穷大（注意是负无穷大而不是无穷小）。负无穷大的值必须到达最小堆的根节点处，此时我们再调用 extractMin()来移除该节点。
4. extractMin()：从最小堆中删除根节点。时间复杂度为 O(logn)，因为删除后还必须通过调用 heapify()来维护 heap 属性。
5. decreaseKey()：减少某个关键字到指定的值。时间复杂度为 O(logn)。一个节点减少后必然依旧小于子节点；如果一个节点减少后的值仍然大于其父节点，那么我们不需要做任何事情。否则，需要遍历来修复冲突的 heap 属性。

```c++
// C++程序 以小顶堆为例演示常见的二叉堆操作
#include<iostream>
using namespace std;
// 用来交换两个整数的函数的原型
void swap(int* x, int* y);
// 定义小顶堆的类
class MinHeap {
    int* heap_arr; // 指针指向一个数组，该数组存放堆中所有元素
    int capacity; // 堆的理论最大容量
    int heap_size; // 堆中当前元素数量
public:
    MinHeap(int capacity); // 构造函数
    void MinHeapify(int); // 以给定索引的元素为根，堆起来一个满足小顶堆属性的子树
    int parent(int i) { return (i - 1) / 2; } // 由已知节点的地址，找其父节点地址
    int left(int i) { return (2 * i + 1); } // 由已知节点找左孩子
    int right(int i) { return (2 * i + 2); } // 由已知节点找右孩子
    int extractMin(); // 拿掉小顶并更新堆，返回拿走的小顶
    void decreaseKey(int i, int new_val); // 将索引i处的堆节点键值减少为新值
    int getMin() { return heap_arr[0]; } // 返回当前的小顶上的值
    void deleteKey(int i); // 删除堆中索引i处的节点
    void insertKey(int k); // 向堆中插入新键k
};

MinHeap::MinHeap(int cap) { // 构造函数。给定容量，建立一个小顶堆
    heap_size = 0; // 当前大小
    capacity = cap; // 最大容量
    heap_arr = new int[cap]; // 初始化数组
}

void MinHeap::insertKey(int k) {
    if (heap_size == capacity) { // 堆已满
        cout << "\nOverflow: Could not insertKey\n";
        return;
    }
    heap_size++;
    int i = heap_size - 1;
    heap_arr[i] = k;
    while (i != 0 && heap_arr[parent(i)] > heap_arr[i]) {
        swap(&heap_arr[i], &heap_arr[parent(i)]);
        i = parent(i);
    }
}

void MinHeap::decreaseKey(int i, int new_val) {
    heap_arr[i] = new_val;
    while (i != 0 && heap_arr[parent(i)] > heap_arr[i]) {
        swap(&heap_arr[i], &heap_arr[parent(i)]);
        i = parent(i);
    }
}

int MinHeap::extractMin() {
    if (heap_size <= 0) return INT_MAX;
    if (heap_size == 1) {
        heap_size--;
        return heap_arr[0];
    }
    int root = heap_arr[0];
    heap_arr[0] = heap_arr[heap_size - 1]; // 把小顶换成最后即较大的元素
    heap_size--;
    MinHeapify(0);
    return root;
}

void MinHeap::deleteKey(int i) {
    decreaseKey(i, INT_MIN);
    extractMin();
}

void MinHeap::MinHeapify(int i) {
    // 若i比子孙大，则小顶堆失去稳定性，需要让浊者下沉（此方法假定子树已经符合作为堆的条件）
    int lindex = left(i);
    int rindex = right(i);
    int smallest = i; // 根左右一家人里最小的那个
    if (lindex < heap_size && heap_arr[lindex] < heap_arr[i]) smallest = lindex;
    if (rindex < heap_size && heap_arr[rindex] < heap_arr[smallest]) smallest = rindex;
    if (smallest != i) {
        swap(&heap_arr[i], &heap_arr[smallest]); // 交换根和孩子，轻者上浮，浊者下沉
        MinHeapify(smallest); // 对原来存放最小元素现在存放根元素的索引进行递归调用，注意是对地址
    }
}

void swap(int* x, int* y) {
    int temp = *x;
    *x = *y;
    *y = temp;
}

int main() {
    MinHeap h(11);
    h.insertKey(3);
    h.insertKey(2);
    h.deleteKey(1);
    h.insertKey(15);
    h.insertKey(5);
    h.insertKey(4);
    h.insertKey(45);
    /*
        2
      4   5
    15 45
    */
    cout << h.extractMin() << endl; // 输出：2
    cout << h.getMin() << endl; // 输出：4
    h.decreaseKey(2, 1);
    cout << h.getMin() << endl; // 输出：1
    return 0;
}
```
