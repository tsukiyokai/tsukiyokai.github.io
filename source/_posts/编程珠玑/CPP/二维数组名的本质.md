## 写在开头，数组名是指针吗？

数组名不是指针，而是一个地址**常量**，可以使用指针操作地址，进而操作数组。

下文之所以说二维数组名是指针，第一，只是想说数组名本质是个地址，但是借指针讲更舒服。第二，指针在解释步长的时候比较方便。何况指针也可以是泛化的概念。你 LeetCode 刷双指针题的时候经常设两个 counter 就能开始遍历了，严格来说也不叫指针，但你能说你解题用的不是双指针的思想？搞清学习的重点，其它不太重要的自己心里门儿清就行了，别那么锱铢必较。

再说一遍，数组名不是指针，但我有时候就是想叫它指针。

## 二维数组名：指向二维数组每一行的指针

众所周知一维数组名是数组首元素的地址。那么二维数组名是否是二维数组首元素的地址呢？

定义一个 2 行 3 列的二维数组

```c++
int a[2][3] = {
    1, 2, 3,
    4, 5, 6
};
```

首先，必须明确二维数组名 a 肯定是一个指针（或直接理解为地址）。接下来我们才能探讨这个指针的含义。
一维数组名是一个指向首元素的指针，我们是通过指针步长来判断的（即：指针进行+1 所跳过的字节数）。
所以自然想到也来判断二维数组名作为一个指针的步长。

**程序示例**

```c++
#include <iostream>
using namespace std;

int main() {
    int a[2][3] = { // 二维数组名肯定是一个指针
        1, 2, 3,
        4, 5, 6
    };
    cout << a << endl; // 指针的地址
    cout << a + 1 << endl; // 指针加一后的地址
    return 0;
}
```

**运行输出**

```
0019FEC4
0019FED0
请按任意键继续. . .
```

**结果分析**

将输出的 2 个十六进制地址换算成十进制表示
1703620
1703632

发现了吗，地址相差 12 字节，也就是步长为 12 字节。而我们创建的二维数组每一行有三个 int 整数，每个 int 整数正好 4 个字节，3×4=12，而12 正是一整行的字节数。所以可以得出结论，二维数组名是一个指向二维数组每一行的指针。

## 二维数组名取地址：指向整个二维数组的指针

如何得到指代整个二维数组的指针？仍然从步长出发思考。

**程序示例**

```c++
#include <iostream>
using namespace std;

int main() {
    int a[2][3] = {
        1, 2, 3,
        4, 5, 6
    };
    cout << &a << endl; // 二维数组名取地址也是一个指针
    cout << &a + 1 << endl; // 指针加一后的地址
    return 0;
}
```

**运行输出**

```
0019FEC4
0019FEDC
请按任意键继续. . .
```

**结果分析**

再次将输出的 2 个十六进制地址换算成十进制表示
1703620
1703644

步长为 24 字节，说明+1 之后指针跳过了整个二维数组，根据类似的分析方法就可以得出结论，对二维数组名取地址，得到的是指向整个二维数组的指针。

## 二维数组名的总结

1. `二维数组名`是指向行的；`&二维数组名`是指向整个数组的。
2. `*二维数组名[i]`指向 i 行的首元素（注意是 i 行而不是第 i 行）；`&二维数组名[j]`指向整个 j 行。

## 二维数组的指针表示法

`*(a+i)`等价于`a[i]`。左边是是先跳 i 行再间接寻址，右边直接就表示 i 行的首元素地址。
更一般地，`a[i][j]`等价于`*(*(a+i)+j)`。

**程序示例**

```c++
#include <iostream>
using namespace std;

int main() {
    /*
    16 18 20
    25 26 27
    */

    int nums[2][3] = { {16, 18, 20}, {25, 26, 27} };
    cout << *(*nums) << endl; // nums[0][0] = 16
    cout << *(*nums + 1) << endl; // nums[0][1] = 18
    cout << *(*nums + 2) << endl; // nums[0][2] = 20
    cout << *(*(nums + 1)) << endl; // nums[1][0] = 25
    cout << *(*(nums + 1) + 1) << endl; // nums[1][1] = 26
    cout << *(*(nums + 1) + 2) << endl; // nums[1][2] = 27

    return 0;
}
```

| Pointer Notation     | Array Notation | Value |
| -------------------- | -------------- | ----- |
| `*(*nums)`           | nums[0][0]     | 16    |
| `*(*nums + 1)`       | nums[0][1]     | 18    |
| `*(*nums + 2)`       | nums[0][2]     | 20    |
| `*(*(nums + 1))`     | nums[1][0]     | 25    |
| `*(*(nums + 1) + 1)` | nums[1][1]     | 26    |
| `*(*(nums + 1) + 2)` | nums[1][2]     | 27    |

## 二维数组名部分碰到的一个 bug

```c++
#include <iostream>
using namespace std;

int main() {
    int a[2][3] = {
        1, 2, 3,
        4, 5, 6
    };

    cout << &a << endl;
    cout << &(a + 1) << endl; // 此行有误，（E0158）表达式必须为左值或函数指示符
    cout << &a + 1 << endl;

    return 0;
}
```

**分析**

解释一下，第一次写的是`&(a + 1)`，报错后改成下面那行即`&a + 1`就正确了。
想了想原因。因为取地址操作符&必须作用于一个左值，而(a+1)明显是一个表达式，不是一个合法的左值，于是就报错了。

左值右值这对概念是从 C 编程中沿袭来的。在 C 中，左值可以出现在等号左边或右边，右值则是只能出现在等号右边。
右值就是一个临时变量（没有在程序中申明），只有临时的地址空间；左值是在程序中具体定义了的，有其地址空间。
换句话说，使用取地址符&对某个值取地址，左值能够得到地址，而右值不能得到地址！
