## Section 1 -- Introduction To Binary Trees

一棵二叉树有许多节点构成，其中每个节点包含一个左指针，一个右指针和一个数据元素。根指针指向树中位置最高的节点。左右指针递归地指向两侧的较小“子树”。空指针表示没有元素的二叉树，即空树。正式的递归定义是：二叉树或是空的（由空指针表示），或是由单个节点组成，其中左右指针皆递归地指向一棵二叉树。

![a drawing of a little binary tree](http://cslibrary.stanford.edu/110/binarytree.gif)

“二叉查找树”或”有序二叉树“是二叉树的一种类型，其中每个节点按如下顺序排列：对于每个节点，其左子树中的所有元素都小于或等于该节点，其右子树中的所有元素都大于该节点。上图展示的就是一棵二叉查找树，根节点是5，其左子树上的节点1 3 4均<=5，其右子树上的节点6 9均>5。递归地，每棵子树也必须满足二分查找树的约束：在1 3 4子树中，3是根，1<=3，4>3。注意这里的准确描述：a "binary search tree" is different from a "binary tree". 树底部边缘的节点具有空子树，称为叶节点（1 4 6），其他节点为内部节点（3 5 9）。

### Binary Search Tree Niche

二叉查找树的插入和删除速度很快，平均时间复杂度为LgN。因此二叉查找树适合解决字典问题 where the code inserts and looks up information indexed by some key。其中LgN是平均情况，特定树可能会变慢，这取决于它的形状。

### Strategy

本文中的某些问题使用普通的二叉树，而有些则使用二叉查找树。无论如何，问题将专注于指针和递归的结合上。

对于每个问题，有两件事要理解...

- 组成树的节点/指针结构，以及对其进行操作的代码。

- 在树上进行迭代的算法，通常是递归。

通常来说，考虑一个二叉树问题时，绘制一些小树来考虑各种情况是个好主意。

### Lookup()

给定一个二叉查找树和一个目标值，搜索树以查看它是否包含目标。 `lookup()`代码的基本模式出现在许多递归树算法中：处理树为空的基本情况，处理当前节点，然后使用递归处理子树。如果树是二叉查找树，则通常在节点上进行某种小于测试，以决定递归是左移还是右移。

```c++
/*
 Given a binary tree, return true if a node
 with the target data is found in the tree. Recurs
 down the tree, chooses the left or right
 branch by comparing the target to each node.
*/
static int lookup(struct node* node, int target) {
    // 1. Base case == empty tree
    // in that case, the target is not found so return false
    if (node == NULL) {
        return false;
    }
    else {
        // 2. see if found here
        if (target == node->data) return true;
        else {
            // 3. otherwise recur down the correct subtree
            if (target < node->data) return(lookup(node->left, target));
            else return(lookup(node->right, target));
        }
    }
}
```

lookup()算法也可以写为使用while循环来遍历树的，而我们的版本使用递归来帮助您解决以下需要递归的问题。

### Pointer Changing Code

指针密集型代码（pointer intensive code）存在一个常见问题：如果函数需要更改传递给它的指针参数之一，该怎么办？例如，下面的`insert()`函数可能要更改根指针。在C和C ++中，一种解决方案是使用指向指针的指针即二级指针（也称为“引用参数”）。这是一种很好的技术，但是在这里，我们将使用一种更简单的技术：希望更改传递给它的指针的函数会将指针的新值返回给调用者。调用者负责使用新值。假设我们有一个`change()`函数可以更改根目录，那么对`change()`的调用将如下所示：

```c++
root = change(root);
```

我们采用`change()`返回的值，并将其用作root的新值。这种结构有点尴尬，但是避免了使用会使一些C和C ++程序员感到困惑的引用参数，而Java根本没有引用参数。这使我们可以专注于递归而不是指针的机制。(For lots of problems that use reference parameters, see CSLibrary #105, Linked List Problems, http://cslibrary.stanford.edu/105/).

### Insert()

给定二叉查找树和一个数字，将具有给定数字的新节点插入树中的正确位置。`insert()`代码类似于`lookup()`，但是复杂之处在于它修改了树形结构。如上所述，`insert()`返回新的树指针以用于其调用者。在这棵树上用数字5调用`insert()`...

```
    2
   / \
  1   10
```

返回树...

```
    2
   / \
  1   10
     /
    5
```

此处显示的解决方案引入了一个`newNode()`辅助函数，该函数构建单个节点。边界条件类似于`lookup()`中的结构——每个调用都检查NULL情况，查看当前节点（the node at hand），然后根据需要向下递归左或右子树。

> base case：边界条件，基准条件，递归出口。

```c++
/*
 Helper function that allocates a new node
 with the given data and NULL left and right
 pointers.
*/
struct node* NewNode(int data) {
    struct node* node = new(struct node);    // "new" is like "malloc"
    node->data = data;
    node->left = NULL;
    node->right = NULL;

    return(node);
}
```

```c++
/*
 Give a binary search tree and a number, inserts a new node
 with the given number in the correct place in the tree.
 Returns the new root pointer which the caller should
 then use (the standard trick to avoid using reference
 parameters).
*/
struct node* insert(struct node* node, int data) {
    // 1. If the tree is empty, return a new, single node
    if (node == NULL) {
        return(NewNode(data));
    }
    else {
        // 2. Otherwise, recur down the tree
        if (data <= node->data) node->left = insert(node->left, data);
        else node->right = insert(node->right, data);

        return(node); // return the (unchanged) node pointer
    }
}
```

二叉树的形状在很大程度上取决于节点插入的顺序。特别地，如果按递增顺序（1 2 3 4）插入节点，则树节点仅向右生长，从而导致二叉树退化成链表形状，其中所有左指针均为NULL。如果按降序插入节点，则也会发生类似的情况。链表形状破坏了二叉树lg(N)的性能。我们并不会在这里解决该问题，而是将重点放在指针和递归上。

### 1. build123()

只需一点指针操作，这是一个非常基本的问题。（如果您已经熟悉了指针则可以跳过此问题。）编写代码，以构建以下简易的1-2-3二叉查找树...

```
    2
   / \
  1   3
```

用三种不同的方式编写代码...

- a: 调用newNode()三次，使用三个指针变量
- b: 调用newNode()三次，只使用一个指针变量
- c: 调用insert()三次，传递根指针来建立树

```c++
// call newNode() three times
struct node* build123a() {
    struct node* root = newNode(2);
    struct node* lChild = newNode(1);
    struct node* rChild = newNode(3);
    root->left = lChild;
    root->right = rChild;

    return(root);
}

// call newNode() three times, and use only one local variable
struct node* build123b() {
    struct node* root = newNode(2);
    root->left = newNode(1);
    root->right = newNode(3);

    return(root);
}

/*
 Build 123 by calling insert() three times.
 Note that the '2' must be inserted first.
*/
struct node* build123c() {
    struct node* root = NULL;
    root = insert(root, 2);
    root = insert(root, 1);
    root = insert(root, 3);
    return(root);
}
```

### 2. size()

此问题说明了简单的二叉树遍历。给定一个二叉树，计算该树中的节点数。

```c++
/*
 Compute the number of nodes in a tree.
*/
int size(struct node* node) {
    if (node == NULL) {
        return(0);
    }
    else {
        return(size(node->left) + 1 + size(node->right));
    }
}
```

### 3. maxDepth()

给定一棵二叉树，计算其“最大深度”——沿着从根节点到最远叶节点的最长路径的节点数。空树的maxDepth为0，第一页上树的maxDepth为3。

```c++
/*
 Compute the "maxDepth" of a tree -- the number of nodes along
 the longest path from the root node down to the farthest leaf node.
*/
int maxDepth(struct node* node) {
    if (node == NULL) {
        return(0);
    }
    else {
        // compute the depth of each subtree
        int lDepth = maxDepth(node->left);
        int rDepth = maxDepth(node->right);
        // use the larger one
        if (lDepth > rDepth) return(lDepth + 1);
        else return(rDepth + 1);
    }
}
```

### 4. minValue()

给定一个非空的二叉查找树(有序二叉树)，返回在该树中找到的最小数据值。注意，没有必要搜索整个树。maxValue()函数在结构上与这个函数非常相似。这可以通过递归或简单的while循环来解决。

```c++
/*
 Given a non-empty binary search tree,
 return the minimum data value found in that tree.
 Note that the entire tree does not need to be searched.
*/
int minValue(struct node* node) {
    struct node* current = node;
    // loop down to find the leftmost leaf
    while (current->left != NULL) {
        current = current->left;
    }

    return(current->data);
}
```

### 5. printTree()

给定一个二叉查找树(又名“有序二叉树”)，遍历节点，以递增的顺序将它们打印出来。所以这棵树…

```
       4
      / \
     2   5
    / \
   1   3
```

产生输出“1 2 3 4 5”。这被称为树的中序遍历。

提示：对于每个节点，策略是：向左递归，打印节点数据，向右递归。

```c++
/*
 Given a binary search tree, print out
 its data elements in increasing
 sorted order.
*/
void printTree(struct node* node) {
    if (node == NULL) return;
    printTree(node->left);
    printf("%d ", node->data);
    printTree(node->right);
}
```

### 6. printPostorder()

给定一个二叉树，根据自底向上的“后序”遍历打印出树的节点——在打印节点本身之前，节点的两个子树都会被完全打印出来，每个左子树都会在右子树之前被打印出来。所以这棵树…

```
       4
      / \
     2   5
    / \
   1   3
```

产生输出“1 3 2 5 4”。描述很复杂，但代码很简单。这是一种自下而上的遍历，例如，用于计算表达式树，其中一个节点是一个类似“+”的操作，其子树递归地是“+”的两个子表达式。

```c++
/*
 Given a binary tree, print its
 nodes according to the "bottom-up"
 postorder traversal.
*/
void printPostorder(struct node* node) {
    if (node == NULL) return;
    // first recur on both subtrees
    printTree(node->left);
    printTree(node->right);

    // then deal with the node
    printf("%d ", node->data);
}
```

### 7. hasPathSum()

我们将“根到叶路径”定义为树中从根节点开始向下到叶（没有子节点的节点）的一系列节点。我们会说一棵空树不包含根到叶的路径。例如，下面的树正好有四条从根到叶的路径：

```
              5
             / \
            4   8
           /   / \
          11  13  4
         /  \      \
        7    2      1
```

根到叶的路径：
路径1：5 4 11 7
路径2：5 4 11 2
路径3：5 8 13
路径4：5 8 4 1

对于这个问题，我们将关注这样一条路径的值之和——例如，5-4-11-7路径上的值之和是5+4+11+7=27。

给定一个二叉树和一个和，如果树有一个从根到叶的路径，使得沿路径的所有值相加等于给定的和，则返回true。如果找不到这样的路径，则返回false。（感谢Owen Astrachan提出这个问题。）

```c++
/*
 Given a tree and a sum, return true if there is a path from the root
 down to a leaf, such that adding up all the values along the path
 equals the given sum.
 Strategy: subtract the node value from the sum when recurring down,
 and check to see if the sum is 0 when you run out of tree.
*/
bool hasPathSum(struct node* node, int sum) {
    // return true if we run out of tree and sum==0
    if (node == NULL) {
        return(sum == 0);
    }
    else {
        // otherwise check both subtrees
        int subSum = sum - node->data;
        return(hasPathSum(node->left, subSum) || hasPathSum(node->right, subSum));
    }
}
```

### 8. printPaths()

给定一个二叉树，按照上面定义的那样打印出它的所有根到叶路径。这个问题比看起来要难一些，因为“到目前为止的路径”需要在递归调用之间通信。

提示：再C、C++和Java中，最好的解决方案可能是创建一个递归的辅助函数printPathsRecur(node, int path[]， int pathLen)，其中路径数组传递导致当前调用的节点序列。或者，这个问题可以自底向上解决，每个节点返回其路径列表。这个策略在Lisp语言中非常好用，因为它可以利用内置的列表和映射原语。(感谢Matthias Felleisen提出这个问题。)

给定一棵二叉树，打印出其所有的从根到叶的路径，每行一条。

```c++
/*
 Given a binary tree, print out all of its root-to-leaf
 paths, one per line. Uses a recursive helper to do the work.
*/
void printPaths(struct node* node) {
    int path[1000];
    printPathsRecur(node, path, 0);
}

/*
 Recursive helper function -- given a node, and an array containing
 the path from the root node up to but not including this node,
 print out all the root-leaf paths.
*/
void printPathsRecur(struct node* node, int path[], int pathLen) {
    if (node == NULL) return;

    // append this node to the path array
    path[pathLen] = node->data;
    pathLen++;

    // it's a leaf, so print the path that led to here
    if (node->left == NULL && node->right == NULL) {
        printArray(path, pathLen);
    }
    else {
        // otherwise try both subtrees
        printPathsRecur(node->left, path, pathLen);
        printPathsRecur(node->right, path, pathLen);
    }
}

// Utility that prints out an array on a line.
void printArray(int ints[], int len) {
    for (int i = 0; i < len; i++) {
        printf("%d ", ints[i]);
    }
    printf("\n");
}
```

### 9. mirror()

更改树，以便在每个节点上交换左右指针的角色。

所以这棵树…

```
       4
      / \
     2   5
    / \
   1   3
```

改变为…

```
       4
      / \
     5   2
        / \
       3   1
```

解决方案十分简短，但是非常具有递归特色。碰巧的是，这可以在不更改根节点指针的情况下完成，因此不需要返回新的根结构。或者，如果您不想更改树节点，则可以基于原始树来构造并返回新的镜像树。

```c++
/*
 Change a tree so that the roles of the
 left and right pointers are swapped at every node.
 So the tree...
       4
      / \
     2   5
    / \
   1   3

 is changed to...
       4
      / \
     5   2
        / \
       3   1
*/
void mirror(struct node* node) {
    if (node == NULL) {
        return;
    }
    else {
        struct node* temp;

        // do the subtrees
        mirror(node->left);
        mirror(node->right);

        // swap the pointers in this node
        temp = node->left;
        node->left = node->right;
        node->right = temp;
    }
}
```

### 10. doubleTree()

对于二叉查找树中的每个节点，创建一个新的重复节点，并将该重复节点作为原始节点的左子节点插入。结果树应该仍然是一个二叉查找树。

So the tree...

```
    2
   / \
  1   3
```

is changed to...

```
       2
      / \
     2   3
    /   /
   1   3
  /
 1
```

与前面的问题一样，这可以在不更改根节点指针的情况下完成。提示：自底向上。

```c++
void doubleTree(struct node* node) {
    struct node* oldLeft;

    if (node == NULL) return;

    // do the subtrees
    doubleTree(node->left);
    doubleTree(node->right);

    // duplicate this node to its left
    oldLeft = node->left;
    node->left = newNode(node->data);
    node->left->left = oldLeft;
}
```

### 11. sameTree()

给定两个二叉树，如果它们在结构上相同，则返回true—它们由具有相同值的节点以相同的方式排列而成。（感谢朱莉·泽伦斯基提出这个问题。）

```c++
/*
 Given two trees, return true if they are
 structurally identical.
*/
bool sameTree(struct node* a, struct node* b) {
    // 1. both empty -> true
    if (a == NULL && b == NULL) return(true);
    // 2. both non-empty -> compare them
    else if (a != NULL && b != NULL)
        return(a->data == b->data && sameTree(a->left, b->left) && sameTree(a->right, b->right));
    // 3. one empty, one not -> false
    else return(false);
}
```

### 12. countTrees()

这不是一个普通意义上的二叉树编程问题——它更像是一个碰巧使用二叉树的数学/组合学递归问题。（感谢杰瑞·凯恩提出这个问题。）

假设您正在构建一个N节点的二叉查找树，其值为1..N。有多少结构上不同的二叉查找树来存储这些值?编写一个递归函数，给定不同值的数量，计算存储这些值的结构上唯一的二叉查找树的数量。例如，countTrees(4)应该返回14，因为有14个结构上唯一的二叉查找树存储1、2、3和4。基本情况很简单，递归简短但密集。你的代码不应该构造任何实际的树，因为这只是一个计数的问题。

```c++
/*
 For the key values 1...numKeys, how many structurally unique
 binary search trees are possible that store those keys.
 Strategy: consider that each value could be the root.
 Recursively find the size of the left and right subtrees.
*/
int countTrees(int numKeys) {
    if (numKeys <= 1) return(1);
    else {
        // there will be one value at the root, with whatever remains
        // on the left and right each forming their own subtrees.
        // Iterate through all the values that could be the root...
        int sum = 0;
        int left, right, root;

        for (root = 1; root <= numKeys; root++) { // 对于每一个有可能成为根节点的root来说，（实际上所有值都有可能成为根）
            left = countTrees(root - 1); // 比root小的只有root-1个元素
            right = countTrees(numKeys - root); // 比root大的有N-root个元素

            // number of possible trees with this root == left*right
            sum += left * right; // 左子树有left种形状，右子树有right种树形，每一种左子树候选的形状可以对应right种右子树形状，因此相乘得到每个根节点的所有子树情况
        }

        return(sum);
    }
}
```

### Binary Search Tree Checking (for problems 13 and 14)

接下来的两个问题会使用此背景：给定一个普通的二叉树，请检查该树以确定其是否满足成为二叉查找树的要求。要成为二叉查找树，对于每个节点，其左树中的所有节点必须为<=节点，而其右子树中的所有节点必须为>节点。考虑以下四个示例...

```
a.  5   -> TRUE
   / \
  2   7
  
b.  5   -> FALSE, because the 6 is not ok to the left of the 5
   / \
  6   7

c.   5  -> TRUE
    / \
   2   7
  /
 1

d.   5  -> FALSE, the 6 is ok with the 2, but the 6 is not ok with the 5
    / \
   2   7
  / \
 1   6
```

对于前两种情况，只需将每个节点与其正下方的两个节点进行比较，便可以看到正确的答案。但是，第四种情况表明，检查BST质量的方式可能取决于相隔几层的节点——在这种情况下为5层和6层。

### 13 isBST() -- version 1

假设您有助手函数minValue()和maxValue()，它们从非空树返回min或max int值（参见上面的问题3）。编写一个isBST()函数，如果树是二叉查找树，则返回true，否则返回false。使用辅助函数，不要忘记检查树中的每个节点。如果你的解决方案不是很有效，那也没关系。（感谢Owen Astrachan提出这个问题，并将其与问题14进行比较）

如果二叉树是二叉查找树，则返回true。

```c++
/*
 Returns true if a binary tree is a binary search tree.
*/
int isBST(struct node* node) {
    if (node == NULL) return(true);
    // false if the max of the left is > than us

    // (bug -- an earlier version had min/max backwards here)
    if (node->left != NULL && maxValue(node->left) > node->data)
        return(false);

    // false if the min of the right is <= than us
    if (node->right != NULL && minValue(node->right) <= node->data)
        return(false);

    // false if, recursively, the left or right is not a BST
    if (!isBST(node->left) || !isBST(node->right))
        return(false);

    // passing all that, it's a BST
    return(true);
}
```

### 14. isBST() -- version 2

上面的版本1运行缓慢，因为它多次遍历树的某些部分。更好的解决方案是只对每个节点检查一次。诀窍是编写一个实用程序辅助函数isBSTRecur()向下遍历树，跟踪缩小的最小和最大允许值，只对每个节点查看一次。min和max的初始值应为INT_MIN和INT_MAX——从那里开始变窄。

```c++
/*
 Returns true if the given tree is a binary search tree
 (efficient version).
*/
bool isBST2(struct node* node) {
    return(isBSTUtil(node, INT_MIN, INT_MAX));
}

/*
 Returns true if the given tree is a BST and its
 values are >= min and <= max.
*/
bool isBSTUtil(struct node* node, int min, int max) {
    if (node == NULL) return(true);

    // false if this node violates the min/max constraint
    if (node->data<min || node->data>max) return(false);

    // otherwise check the subtrees recursively,
    // tightening the min or max constraint
    return
        isBSTUtil(node->left, min, node->data) && isBSTUtil(node->right, node->data + 1, max);
}
```

### 15. Tree-List

Tree-List问题是有史以来最大的递归指针问题之一，它也恰好使用二叉树。CLibarary #109 http://cslibrary.stanford.edu/109/ works through the Tree-List problem in detail and includes solution code in C and Java. 这个问题需要理解二叉树、链表、递归和指针。这是个很棒的问题，但也很复杂。

```c++
/*
 TreeList.c

 C code version of the great Tree-List recursion problem.
 See http://cslibrary.stanford.edu/109/ for the full
 discussion and the Java solution.

 This code is free for any purpose.
 Feb 22, 2000
 Nick Parlante nick.parlante@cs.stanford.edu
*/

#include <stdio.h>
#include <stddef.h>
#include <stdlib.h>

/* The node type from which both the tree and list are built */
struct node {
    int data;
    struct node* small;
    struct node* large;
};
typedef struct node* Node;

/*
 helper function -- given two list nodes, join them
 together so the second immediately follow the first.
 Sets the .next of the first and the .previous of the second.
*/
static void join(Node a, Node b) {
    a->large = b;
    b->small = a;
}

/*
 helper function -- given two circular doubly linked
 lists, append them and return the new list.
*/
static Node append(Node a, Node b) {
    Node aLast, bLast;

    if (a == NULL) return(b);
    if (b == NULL) return(a);

    aLast = a->small;
    bLast = b->small;

    join(aLast, b);
    join(bLast, a);

    return(a);
}

/*
 --Recursion--
 Given an ordered binary tree, recursively change it into
 a circular doubly linked list which is returned.
*/
static Node treeToList(Node root) {
    Node aList, bList;

    if (root == NULL) return(NULL);

    /* recursively solve subtrees -- leap of faith! */
    aList = treeToList(root->small);
    bList = treeToList(root->large);

    /* Make a length-1 list ouf of the root */
    root->small = root;
    root->large = root;

    /* Append everything together in sorted order */
    aList = append(aList, root);
    aList = append(aList, bList);

    return(aList);

    /* Create a new node */
    static Node newNode(int data) {
        Node node = (Node)malloc(sizeof(struct node));
        node->data = data;
        node->small = NULL;
        node->large = NULL;
        return(node);
    }

    /* Add a new node into a tree */
    static void treeInsert(Node * rootRef, int data) {
        Node root = *rootRef;
        if (root == NULL) *rootRef = newNode(data);
        else {
            if (data <= root->data) treeInsert(&(root->small), data);
            else treeInsert(&(root->large), data);
        }
    }

    static void printList(Node head) {
        Node current = head;

        while (current != NULL) {
            printf("%d ", current->data);
            current = current->large;
            if (current == head) break;
        }
        printf("\n");
    }

    /* Demo that the code works */
    int main() {
        Node root = NULL;
        Node head;

        treeInsert(&root, 4);
        treeInsert(&root, 2);
        treeInsert(&root, 1);
        treeInsert(&root, 3);
        treeInsert(&root, 5);

        head = treeToList(root);

        printList(head);    /* prints: 1 2 3 4 5  */

        return(0);
    }
```
