## 存储时的 3 个区域

一个存储着数据但处于未运行状态的程序，其存储结构分为三个部分：
（未运行的情况包括它已经被编译过但没有执行——注意编译和运行的差别）

```tex
3: Uninitialized data segment (BSS)
2: Initialized data segment (Data)
1: Text segment (Text)
```

## 运行时的 5 个区域

一个正在运行着的 C 程序占用的内存可以分为 5 个部分。注：从下到上，1 为低地址，5 为高地址。

```tex
5: Stack (Stack) ↓
4: Heap (Heap) ↑
3: Uninitialized data segment (BSS)
2: Initialized data segment (Data)
1: Text segment (Text)
```

5 以上是各种命令行参数和环境变量
5 栈向低地址增长
4 堆向高地址增长
3 由 exec 函数族初始化为 0
1 和 2 由 exec 函数族从程序文件中读入

接下来自底向上依次说明。

## Text 代码段

内存模型中的代码段部分是目标文件或内存中程序的一部分，其中包含可执行指令。

作为一片内存区域，Text 段最好放置在堆或栈的下面，以防堆栈溢出覆盖它。（堆以低地址为自己的底部，所以是向上发生溢出的。如果代码段设置在堆栈上面，就会有被覆盖的危险）

通常来说，文本段是可被共享的，所以内存中只需要有一个副本，就能满足频繁执行的程序的需求（如文本编辑器、C 编译器、shell 等）。此外，文本段一般是只读的，以防内容被意外修改。

## Data 数据段

已初始化的数据段，通常简称为数据段。包含已经由程序员初始化的全局变量和静态变量。例如，任何位置的 static int i = 10 都将存储在数据段中，全局的 int i = 10 也将存储在数据段中。

请注意，数据段不是只读的，因为它可以在运行时更改变量的值。但该段也可以进一步细分为已初始化只读区和已初始化读写区。

## BSS BSS 段

未初始化的数据段通常称为 bss 段，它以一个古老的汇编程序运算符命名，意思是 a block started by symbol（以符号开头的块）。

bss 中的数据的地址从 Data 段的末尾开始增长，bss 包含所有**初始化为零**或在源代码中**没有显式初始化**的全局变量和静态变量。这个段中的数据将会在程序开始执行前由内核初始化为算术 0。bss 段属于静态内存分配，可见它只存在于编译时，不存在于运行时。一旦程序开始运行，在 bss 内存区进入程序的地址空间后，这个段的数据将全部清零，包含 data 和 bss 段的整个区段此时统称为数据区。注意这里清零的意思其实是置零，说清零有点奇怪，因为本来就没有赋初值，何来清空呢？

既然一开始执行程序就要清零 bss 段，那它为何还要存在？原因是 bss 段并不给该段的数据分配空间，它的作用是记录数据所需空间的大小。

举例说明

```c++
// 程序1
int arr[30000];
void main() {
	...
}

// 程序2
int arr[30000] = {1, 2, 3, 4, 5, 6};
void main() {
	...
}
```

编译之后发现程序 2 的可执行文件比程序 1 大得多。这是因为程序 1 的数组 arr 位于 bss 段，bss 不占用可执行文件空间，其内容由操作系统初始化（清零）。而程序 2 位于 data 段，data 段需要占用可执行文件空间，其内容由程序初始化。

一个程序本质上是由 bss 段、data 段、text 段三个组成的。在一个 C 程序编译完成之后正式执行之前，已初始化的全局变量保存在 data 段中，未初始化的全局变量保存在 bss 段中。text 和 data 段都在可执行文件中（在嵌入式系统里一般是固化在镜像文件中），由系统从可执行文件中加载；而 bss 段不在可执行文件中，由系统初始化。

例如一个变量声明为`static int i`或`int j`，那么它将被包含在 bss 段中。

## Heap

堆是动态内存分配行为频发的段。

堆区域开始于 bss 段的结尾处，并从那里扩展到更大的地址。堆区域由 new 和 delete，malloc，realloc 和 free 管理，它们可以使用 brk 和 sbrk 系统调用来调整其大小。堆区域由进程中所有共享库和动态加载的模块共享。

一定要记得使用完后及时释放所申请的空间。

## Stack

栈区域在传统上与堆区域相邻，并以相反的方向增长，当栈指针遇到堆指针时，可用内存耗尽。（借助现代的大型地址空间和虚拟内存技术，堆区和栈区几乎可以放置在任何地方，但它们一般还是沿相反的方向生长。）

我看过的一种比较靠谱的说法是，当初的内存容量小，栈是从地址高位向低位分配，而堆是从低位向高位分配，这样可以最大化压榨内存。

栈区域包含栈数据结构，一种先进后出结构，它通常位于内存的较高部分。在标准的 x86 计算机体系结构上，它朝着地址 0 增长；在其他一些架构上，它则朝着相反的方向增长。栈顶指针寄存器跟踪着栈顶的位置，每次新值入栈时都会对其进行调整。为了完成一次函数调用而推入的一组值称为“栈帧”，栈帧也叫过程活动记录，是编译器用来实现函数调用的一种数据结构。栈帧至少包含一个返回地址。

栈用于存储自动变量以及每次调用函数时保存的信息。每次调用函数时，返回位置的地址和调用者环境的某些信息（例如某些机器寄存器）都保存在栈中。接着新调用的函数在栈上为其自动和临时变量分配空间。这就是 C 中递归函数的工作方式。递归函数每次调用自己时，都会使用一个新的栈帧，这样一组变量就不会干扰来自该函数另一个实例的变量。

注：自动变量就是普通的局部变量，前面不带 static。自动指的是局部变量离开块作用域自动销毁，临时指的是局部变量具有（短暂的）自动存储期限。由此可见，栈区域主要是用来存放函数内的局部变量，在函数执行结束时自动释放。栈内存的分配运算内置于处理器的指令集中，显然效率很高，但是可以分配的内存容量十分有限。

题外话：注意区分内存模型中的栈区域和数据结构中的栈结构。stack 这种 LIFO 结构由于比较复杂的历史和现实因素在国内普遍被翻译为堆栈，实在是有些误人子弟（台湾人背锅），即使在我的母校吉林大学，老师们一个个也乐此不疲地一口一个“堆栈”。学生自己还是要清楚：栈 is 栈，堆 is 堆，由计算机科学技术名词审定委员会编撰完成的《计算机科学技术名词》一书里完全就没有堆栈这个名词。不过话说回来，如果你真的能完全把堆栈和堆这两个东西当成两件事，那喊栈为堆栈也没什么大问题。私下里我还是觉得规范化用语比较好。
