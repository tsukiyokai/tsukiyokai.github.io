二维数组名的本质

发表于 2021-04-18 分类于 编程珠玑 → C++程序设计 阅读次数：

写在开头，数组名是指针吗？

数组名不是指针，而是一个地址常量，可以使用指针操作地址，进而操作数组。

下文之所以说二维数组名是指针，第一，只是想说数组名本质是个地址，但是借指针讲更舒服。第二，指针在解释步长的时候比较方便。何况指针也可以是泛化的概念。你LeetCode刷双指针题的时候经常设两个counter就能开始遍历了，严格来说也不叫指针，但你能说你解题用的不是双指针的思想？搞清学习的重点，其它不太重要的自己心里门儿清就行了，别那么锱铢必较。

再说一遍，数组名不是指针，但我有时候就是想叫它指针。

二维数组名：指向二维数组每一行的指针

众所周知一维数组名是数组首元素的地址。那么二维数组名是否是二维数组首元素的地址呢？

二维数组名的总结

  1. 二维数组名是指向行的；&二维数组名是指向整个数组的。
  2. *二维数组名[i]指向i行的首元素（注意是i行而不是第i行）；&二维数组名[j]指向整个j行。

二维数组的指针表示法

*(a+i)等价于a[i]。左边是是先跳i行再间接寻址，右边直接就表示i行的首元素地址。
更一般地，a[i][j]等价于*(*(a+i)+j)。

程序示例

Pointer Notation Array Notation Value
*(*nums) nums[0][0] 16
*(*nums + 1) nums[0][1] 18
*(*nums + 2) nums[0][2] 20
*(*(nums + 1)) nums[1][0] 25
*(*(nums + 1) + 1) nums[1][1] 26
*(*(nums + 1) + 2) nums[1][2] 27

二维数组名部分碰到的一个bug

分析

解释一下，第一次写的是&(a + 1)，报错后改成下面那行即&a + 1就正确了。
想了想原因。因为取地址操作符&必须作用于一个左值，而(a+1)明显是一个表达式，不是一个合法的左值，于是就报错了。

左值右值这对概念是从C编程中沿袭来的。在C中，左值可以出现在等号左边或右边，右值则是只能出现在等号右边。
右值就是一个临时变量（没有在程序中申明），只有临时的地址空间；左值是在程序中具体定义了的，有其地址空间。
换句话说，使用取地址符&对某个值取地址，左值能够得到地址，而右值不能得到地址！
