二叉树 介绍

树：与数组，链表，堆栈和队列这些线性数据结构不同，树是层次数据结构。

为什么选择树？

  1. 使用树的一个原因可能是因为您要存储的信息自然而然地形成了某种层次结构。例如，计算机上的文件系统。
  2. 树（以某种顺序，例如BST）提供中等程度的访问/搜索（比“链表”更快，比“数组”慢）。
  3. 树提供中等程度的插入/删除（比数组更快，比无序链表慢）。
  4. 与链表一样，与数组不同，树没有结点数上限，因为结点之间是使用指针链接的。

树的主要应用包括

  1. 处理分层数据。
  2. 使信息易于搜索（请参阅遍历树）。
  3. 处理有序的数据列表。
  4. 作为合成数字图像以获得视觉效果的工作流程。
  5. Router algorithms（路由算法）。
  6. 多阶段决策的形式（请参阅商务象棋）。

二叉树的性质

  1. 非空二叉树的第i层最多有2^(i-1)个结点。
  2. 深度为h的非空二叉树最多有2^h-1个结点。

二叉树的种类

  1. 满二叉树/完美二叉树 Perfect Binary Tree

  2. 完全二叉树 Complete Binary Tree

  3. 完满二叉树 Full Binary Tree（没有度为1的结点）

  4. 平衡二叉树 Balanced Binary Tree
    A binary tree is balanced if the height of the tree is O(Log n) where n is the number of nodes.For Example, the AVL tree maintains O(Log n) height by making sure that the difference between the heights of the left and right subtrees is atmost 1. Red-Black trees maintain O(Log n) height by making sure that the number of Black nodes on every root to leaf paths is the same and there are no adjacent red nodes. Balanced Binary Search trees are performance-wise good as they provide O(log n) time for search, insert and delete.

  5. 退化树、病理树 A degenerate (or pathological) tree（在性能上与链表相同）

Section 1 – Introduction To Binary Trees

一棵二叉树有许多节点构成，其中每个节点包含一个左指针，一个右指针和一个数据元素。根指针指向树中位置最高的节点。左右指针递归地指向两侧的较小“子树”。空指针表示没有元素的二叉树，即空树。正式的递归定义是：二叉树或是空的（由空指针表示），或是由单个节点组成，其中左右指针皆递归地指向一棵二叉树。

“二叉查找树”或”有序二叉树“是二叉树的一种类型，其中每个节点按如下顺序排列：对于每个节点，其左子树中的所有元素都小于或等于该节点，其右子树中的所有元素都大于该节点。上图展示的就是一棵二叉查找树，根节点是5，其左子树上的节点1 3 4均<=5，其右子树上的节点6 9均>5。递归地，每棵子树也必须满足二分查找树的约束：在1 3 4子树中，3是根，1<=3，4>3。注意这里的准确描述：a “binary search tree” is different from a “binary tree”. 树底部边缘的节点具有空子树，称为叶节点（1 4 6），其他节点为内部节点（3 5 9）。

Binary Search Tree Niche

二叉查找树的插入和删除速度很快，平均时间复杂度为LgN。因此二叉查找树适合解决字典问题 where the code inserts and looks up information indexed by some key。其中LgN是平均情况，特定树可能会变慢，这取决于它的形状。

Strategy

本文中的某些问题使用普通的二叉树，而有些则使用二叉查找树。无论如何，问题将专注于指针和递归的结合上。

对于每个问题，有两件事要理解…

  • 组成树的节点/指针结构，以及对其进行操作的代码。

  • 在树上进行迭代的算法，通常是递归。

通常来说，考虑一个二叉树问题时，绘制一些小树来考虑各种情况是个好主意。

Lookup()

给定一个二叉查找树和一个目标值，搜索树以查看它是否包含目标。 lookup()代码的基本模式出现在许多递归树算法中：处理树为空的基本情况，处理当前节点，然后使用递归处理子树。如果树是二叉查找树，则通常在节点上进行某种小于测试，以决定递归是左移还是右移。

lookup()算法也可以写为使用while循环来遍历树的，而我们的版本使用递归来帮助您解决以下需要递归的问题。

二叉树的形状在很大程度上取决于节点插入的顺序。特别地，如果按递增顺序（1 2 3 4）插入节点，则树节点仅向右生长，从而导致二叉树退化成链表形状，其中所有左指针均为NULL。如果按降序插入节点，则也会发生类似的情况。链表形状破坏了二叉树lg(N)的性能。我们并不会在这里解决该问题，而是将重点放在指针和递归上。

二叉树 遍历

正文

与那些只能以一种逻辑方式遍历的线性数据结构不同，树可以用多种方式遍历：

主要分为DFS深度优先和BFS广度优先两种。深度优先指前中后序遍历，广度优先指层次遍历level order。

  1. 先根遍历，用于：创建一棵树的拷贝、前缀表达式（波兰表示法）。

为什么前缀表达式十分有用？why prefix expressions are useful?

波兰表示法（Polish notation），是一种逻辑、算术和代数表示方法，其特点是操作符置于操作数的前面，因此也称做前缀表示法。如果操作符的元数（arity）是固定的，则语法上不需要括号仍然能被无歧义地解析。波兰记法是波兰数学家扬·武卡谢维奇1920年代引入的，用于简化命题逻辑。

  1. 中根遍历，用于：对一棵二叉查找树（BST）进行Inorder可得到非降序排列。

  2. 后根遍历，用于：删除树、后缀表达式（逆波兰表示法）。

逆波兰表示法（Reverse Polish notation，RPN），是一种是由波兰数学家扬·武卡谢维奇1920年引入的数学表达式方式，在逆波兰记法中，所有操作符置于操作数的后面，因此也被称为后缀表示法。逆波兰记法不需要括号来标识操作符的优先级。
